# Yandex Practicum Go Structures Sprint 5 Final

Итоговый проект обучения языку **Go**, посвящённый использованию **структур, embedding, методов и интерфейсов**.

## Описание

Данный проект является улучшенной версией предыдущего проекта и демонстрирует применение более продвинутых возможностей языка Go.

Основная цель проекта — переработать существующую логику приложения с использованием **структур**, **встраивания (embedding)**, **методов** и **интерфейсов**, чтобы сделать код более модульным, расширяемым и соответствующим идиоматическому стилю Go.

Проект представляет собой апгрейд предыдущей версии приложения, в котором основная логика была реализована без активного использования этих возможностей языка.

## Функционал

Реализована работа с посылками и управление их состояниями:

* **Add** — добавление новой посылки
* **Get** — получение посылки по номеру
* **GetByClient** — получение всех посылок клиента
* **SetStatus** — изменение статуса посылки
* **SetAddress** — изменение адреса (доступно только для посылок со статусом `Registered`)
* **Delete** — удаление посылки (доступно только для посылок со статусом `Registered`)

## Технологии

* **Go** — основной язык разработки
* **Структуры (structs)** — для описания моделей данных
* **Методы (methods)** — для реализации логики работы с сущностями
* **Embedding** — для переиспользования поведения и композиции структур
* **Интерфейсы (interfaces)** — для абстракции и гибкости архитектуры

## Ключевые концепции

В проекте применяются основные возможности Go для построения более чистой архитектуры:

* Использование **структур** для представления доменных моделей
* Реализация **методов** для инкапсуляции логики
* Применение **embedding** для композиции структур
* Использование **интерфейсов** для абстрагирования поведения
* Разделение логики и данных для улучшения читаемости и масштабируемости

## Отличия от предыдущего проекта

По сравнению с предыдущей версией проекта:

* Добавлено использование **структур вместо простых типов**
* Реализованы **методы для работы с сущностями**
* Использован **embedding** для композиции структур
* Добавлены **интерфейсы для абстракции логики**
* Улучшена **архитектура и читаемость кода**

## Примечание

Проект демонстрирует практическое применение ключевых концепций Go:

* использование **композиции вместо наследования**
* разделение ответственности между структурами
* построение гибкой архитектуры через **интерфейсы**

Основная цель — показать понимание и применение **структур, embedding, методов и интерфейсов** в реальном приложении.
